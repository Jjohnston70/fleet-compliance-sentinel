## THE VISION

Build a self-contained training module inside FCS where fleet operators can assign, deliver, track, and test employee training -- all without depending on external platforms like OTIS. Training content is generated from the knowledge base we've already built (ERG 2024, CFR docs, PHMSA modules, competitive intel). Content lives as markdown, gets converted to slide decks via the Electron deck engine, and includes built-in assessments. When an employee completes training and passes the test, compliance status auto-updates. Fully self-contained per client org.

This is not a one-off hazmat feature. It's a training platform that starts with hazmat (because we have the content) and expands to any compliance topic: defensive driving, drug/alcohol awareness, DVIR procedures, cargo securement, new hire onboarding, etc.

## PHASE 3: Assessment Engine

### 3.1 -- Define assessment question schema

Each training module includes assessment questions embedded in the markdown. The assessment engine parses these and presents them after the deck is completed.

**Question types to support:**
- Multiple choice (single answer)
- Multiple select (check all that apply)
- True/False
- Scenario-based (situation description + question)

**Question format in markdown:**
```markdown
## Assessment Questions

### Q1
**Type**: multiple_choice
**Question**: When shipping a Class 3 Flammable Liquid, which placard is required on a highway vehicle carrying over 1,000 lbs?
**A**: FLAMMABLE GAS
**B**: FLAMMABLE (correct)
**C**: COMBUSTIBLE
**D**: DANGEROUS
**Explanation**: Class 3 materials require the FLAMMABLE placard per 49 CFR 172.504, Table 2 when the gross weight exceeds 1,000 lbs.
**CFR Reference**: 49 CFR 172.504

### Q2
**Type**: true_false
**Question**: A driver transporting hazardous materials must have shipping papers within arm's reach at all times while driving.
**Answer**: true
**Explanation**: Per 49 CFR 177.817(e), shipping papers must be readily accessible to the driver or within the driver's reach when restrained by the seat belt.
```

- [ ] Create `tools/parse-assessment.py` -- extracts questions from training markdown into assessment JSON
- [ ] Define `TrainingAssessment` data model (questions, answers, scoring, passing threshold)
- [ ] Store assessments alongside deck JSON (one assessment per training module)

### 3.2 -- Build assessment UI component

- [ ] Build `TrainingAssessment.tsx` component:
  - Presented after trainee completes all slides in a module
  - Randomizes question order
  - Shows one question at a time with progress indicator
  - No back-navigation (prevent answer shopping)
  - Scores on completion, shows pass/fail with passing threshold from frontmatter
  - On pass: triggers compliance record update (auto-creates hazmat_training_record)
  - On fail: shows which questions were wrong with explanations, offers retake
  - Records attempt history (date, score, pass/fail) for audit trail
  - Locks retakes for configurable cooldown period (default: 24 hours)

### 3.3 -- Assessment API endpoints

```
POST   /api/v1/training/{module_code}/assessment/submit   -- Submit answers, get scored
GET    /api/v1/training/{module_code}/assessment/attempts  -- Attempt history for employee
GET    /api/v1/training/{employee_id}/assessments/summary  -- All assessment results
```
