# Daily Operations Standup

**Schedule:** Run every weekday at 8:00 AM MT  
**Duration:** ~5 minutes  
**Output:** Slack message to #ops-daily

---

## Objective

Generate a comprehensive daily operations report covering:
- Critical Sentry errors requiring attention
- HubSpot pipeline health (stale deals, upcoming calls)
- Site performance indicators
- Action items for the day

---

## Step 1: Check Sentry for Critical Issues

**Tool:** Sentry MCP  
**Action:** `Sentry:search_issues`

**Parameters:**
```json
{
  "query": "is:unresolved level:error",
  "project": "pipeline-punks-nextjs",
  "statsPeriod": "24h",
  "sort": "freq"
}
```

**Analysis:**
- Count total unresolved errors
- Identify new issues (first seen < 24h)
- Flag any critical errors (level: fatal or error with >10 events)
- Group by error type/location

**Output Format:**
```
**🔥 Sentry Status:**
- Total unresolved: [count]
- New in last 24h: [count]
- Critical issues: [count if any]

[If critical issues exist:]
⚠️ NEEDS ATTENTION:
- [Issue title] - [event count] events - [link]
```

---

## Step 2: HubSpot Pipeline Health Check

**Tool:** HubSpot MCP  
**Action:** `HubSpot:search_crm_objects`

### 2a. Check Stale Deals

**Parameters:**
```json
{
  "object_type": "deals",
  "filters": {
    "dealstage": ["identify", "assess", "map", "chart"],
    "hs_lastmodifieddate": "<7 days ago"
  },
  "properties": [
    "dealname",
    "dealstage", 
    "amount",
    "hs_lastmodifieddate",
    "hubspot_owner_id"
  ],
  "limit": 20
}
```

**Analysis:**
- Calculate days since last activity for each deal
- Prioritize by stage (Chart > Map > Assess > Identify)
- Flag deals >7 days without activity

**Output Format:**
```
**📊 HubSpot Pipeline:**
- Active deals: [count]
- Stale deals (>7 days): [count]

[If stale deals exist:]
⏰ NEEDS FOLLOW-UP:
- [Company] - [Stage] stage - [X] days idle - $[amount]
```

### 2b. Check Today's Scheduled Calls

**Tool:** Google Calendar MCP  
**Action:** `Google Calendar:gcal_list_events`

**Parameters:**
```json
{
  "timeMin": "today 00:00",
  "timeMax": "today 23:59",
  "q": "Direction Protocol OR Identify OR Assess OR Map OR Chart"
}
```

**Output Format:**
```
**📅 Today's Calls:**
[If calls exist:]
- [Time] - [Call type] with [Company] ([HubSpot deal stage])

[If no calls:]
- No Direction Protocol calls scheduled
```

---

## Step 3: Site Health Check

**Tool:** Sentry MCP  
**Action:** `Sentry:search_events`

**Parameters:**
```json
{
  "query": "event.type:error",
  "project": "pipeline-punks-nextjs",
  "statsPeriod": "24h"
}
```

**Analysis:**
- Calculate error rate (events per hour)
- Compare to baseline (normal: <10/hour)
- Flag if error rate is elevated

**Output Format:**
```
**🌐 Site Health:**
- Error rate: [X] events/hour [OK/⚠️ ELEVATED]
- Uptime: [Based on error patterns]

[If elevated:]
⚠️ Higher than normal error rate - investigate Sentry dashboard
```

---

## Step 4: Generate Action Items

**Logic:**
Based on findings above, create prioritized action list:

1. **Critical Sentry errors** → Immediate triage
2. **Stale deals in Chart stage** → Send quotes today
3. **Stale deals in Map/Assess** → Schedule follow-up calls
4. **Elevated error rate** → Run diagnostic check

**Output Format:**
```
**✅ Action Items:**
1. [Highest priority action]
2. [Second priority action]
3. [Third priority action]

[If no critical issues:]
✅ All systems normal - focus on proactive outreach
```

## Notes

- **First Run:** Establish baseline metrics (normal error rate, typical deal velocity)
- **Customize:** Adjust thresholds based on your actual patterns
- **Expand:** Add more checks as you connect additional tools (Datadog, etc.)
- **Alerts:** If critical issues detected, also send to #tech-alerts channel

---

## Maintenance

**Weekly Review:** 
- Check if thresholds need adjustment (error rate baseline, stale deal days)
- Review which deals are repeatedly flagged as stale (may need disqualification)
- Tune notification verbosity (too noisy? too quiet?)

**Monthly Update:**
- Add new HubSpot deal stages as Direction Protocol evolves
- Add new Sentry projects as you launch additional sites/services
- Refine action item priority logic based on what actually gets addressed
