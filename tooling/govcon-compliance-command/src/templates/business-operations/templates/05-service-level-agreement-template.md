Purpose

This Service Level Agreement (SLA) Template provides [COMPANY_LEGAL_NAME] with a standardized format for defining service levels, performance metrics, and responsibilities for services delivered to clients. This template ensures clear expectations and accountability.

Scope

This template applies to all [COMPANY_NAME] managed services and ongoing support agreements. It applies to all personnel involved in service delivery and client management.

Governance and Cross-Package References

This SLA template integrates security, compliance, and data protection requirements into service level agreements.

Security SLA Requirements:
All SLAs must reference security requirements per security-governance skill
All SLAs must include data protection terms per data-handling-privacy skill
All SLAs must include incident response and breach notification timelines per internal-compliance skill (Incident Response Plan) and data-handling-privacy skill (Breach Notification Procedure)
All SLAs must include compliance expectations (HIPAA, FedRAMP, SOC 2, etc.) per security-governance skill

Cross-Package Integration:
security-governance skill: Master governance document; SLAs must align with security framework
data-handling-privacy skill: Data classification and handling requirements for client data
data-handling-privacy skill (Records Management Policy): Data retention requirements for client data
internal-compliance skill (Incident Response Plan): Incident response timelines and procedures

Client-Specific Requirements:
SLAs must document client-specific regulatory obligations (HIPAA, FedRAMP, ITAR, etc.)
SLAs must document client-specific security requirements (encryption, access controls, monitoring)
SLAs must document client-specific retention requirements per data-handling-privacy skill (Records Management Policy)
Client-specific requirements override generic [COMPANY_ABBREVIATION] policies when more stringent

For security requirements, see security-governance skill.
For data handling requirements, see data-handling-privacy skill.
For incident response requirements, see internal-compliance skill (Incident Response Plan).
