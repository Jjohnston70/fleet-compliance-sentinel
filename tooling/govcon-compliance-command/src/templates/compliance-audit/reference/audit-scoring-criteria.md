# AUDIT SCORING CRITERIA
Reference Guide for Compliance Assessments

**Document Purpose:** Unified scoring rubric for compliance audits and assessments
**Version:** 1.0

**Owner:** Compliance Audit Team

## OVERVIEW

This reference document provides standardized scoring criteria for conducting compliance audits, package maturity assessments, and comprehensive compliance reviews. Use these criteria to ensure consistent, objective, and repeatable assessments across all compliance documentation and implementations.

---

## OVERALL MATURITY RATING SCALE

### 10-Point Scale

| Score | Rating | Description | Characteristics |
|-------|--------|-------------|-----------------|
| **9-10** | **EXCELLENT** | Exceptional maturity, audit-ready | - Comprehensive documentation across all areas<br>- Strong compliance framework alignment<br>- Clear cross-package integration<br>- Extensive audit-ready evidence<br>- Realistic operational model<br>- Only minor enhancements needed<br>- Rivals organizations 10x the size |
| **7-8** | **STRONG** | Mature, targeted improvements needed | - Mature documentation with good coverage<br>- Good compliance alignment with some gaps<br>- Most cross-package dependencies documented<br>- Mostly audit-ready<br>- Some targeted improvements required<br>- Solid foundation with enhancement opportunities |
| **5-6** | **ADEQUATE** | Functional, significant work required | - Basic documentation coverage<br>- Partial compliance alignment<br>- Inconsistent cross-package integration<br>- Requires preparation for audit<br>- Significant improvements needed<br>- Functional but not competitive |
| **3-4** | **DEVELOPING** | Major gaps, substantial work needed | - Incomplete documentation<br>- Weak compliance alignment<br>- Missing cross-package integration<br>- Not audit-ready<br>- Major improvements required<br>- Fundamental issues to address |
| **1-2** | **INSUFFICIENT** | Critical deficiencies | - Minimal or no documentation<br>- No compliance framework alignment<br>- No cross-package integration<br>- Not suitable for use<br>- Fundamental rewrites needed<br>- Does not meet minimum standards |

## ASSESSMENT CRITERIA BY DIMENSION

### 1. Completeness

**Excellent (9-10 points):**
- All required policies, procedures, and standards documented
- Supporting documents and templates complete
- No missing elements identified
- Documentation covers all relevant scenarios
- Depth appropriate for organization size and risk profile

**Strong (7-8 points):**
- Most required documents present
- Minor gaps in supporting materials
- Core requirements fully covered
- Some edge cases may need documentation
- Generally comprehensive with targeted additions needed

**Adequate (5-6 points):**
- Basic required documents present
- Several supporting documents missing
- Core requirements mostly covered
- Many scenarios lack documentation
- Functional but incomplete

**Developing (3-4 points):**
- Many required documents missing
- Major gaps in coverage
- Core requirements partially addressed
- Most scenarios lack documentation
- Substantial documentation work required

**Insufficient (1-2 points):**
- Few documents present
- Critical gaps throughout
- Core requirements not addressed
- Scenarios not documented
- Fundamental documentation effort required

### 2. Compliance Alignment

**Excellent (9-10 points):**
- Strong alignment with all applicable frameworks (NIST, ISO, SOC 2, HIPAA, etc.)
- Controls mapped to framework requirements
- Gap analysis documented
- Remediation plans in place
- Language accurate and appropriate (e.g., "aligned" vs "compliant")
- Ready for certification audit when business case supports

**Strong (7-8 points):**
- Good alignment with most frameworks
- Most controls mapped
- Some gaps identified
- Remediation plans mostly defined
- Language generally appropriate
- Near audit-ready with targeted improvements

**Adequate (5-6 points):**
- Partial alignment with frameworks
- Some controls mapped
- Many gaps identified
- Remediation plans incomplete
- Language may overstate capabilities
- Requires preparation before audit

**Developing (3-4 points):**
- Weak alignment with frameworks
- Few controls mapped
- Major gaps throughout
- No remediation plans
- Language may be inaccurate
- Not audit-ready

**Insufficient (1-2 points):**
- No meaningful framework alignment
- Controls not mapped
- Critical gaps across all areas
- No gap analysis or remediation
- Language misleading
- Fundamental work required

## LANGUAGE ACCURACY GUIDELINES

### Compliant vs. Aligned

**Use "Aligned" or "Controls Implemented and Aligned":**
- When controls are implemented but not independently certified
- When framework requirements are met but no audit has occurred
- For frameworks where certification is optional (NIST CSF, GDPR, CCPA)
- To maintain honest representation

**Use "Compliant" only when:**
- Independent certification or audit has been completed
- Regulatory requirement has been formally verified
- Attestation letter or certificate has been issued

### Examples of Appropriate Language

| Framework | Inappropriate | Appropriate |
|-----------|--------------|-------------|
| HIPAA | "HIPAA compliant" | "Controls implemented and aligned with HIPAA; ready for audit when required" |
| SOC 2 | "SOC 2 compliant" | "Controls aligned with SOC 2 Trust Services Criteria; ready for audit when required" |
| ISO 27001 | "ISO 27001 compliant" | "Comprehensive ISMS documented; certification-ready when business case supports" |
| FedRAMP | "FedRAMP certified" | "FedRAMP-adjacent controls at Moderate level" |
| NIST CSF | "NIST compliant" | "Strong alignment with NIST Cybersecurity Framework 2.0" |
| GDPR | "GDPR compliant" | "Strong privacy controls aligned with GDPR requirements" |

## 24/7 COVERAGE REALISM

### Appropriate Language for Small Organizations

**Unrealistic:**
- "24/7 incident detection and response capability" (solo founder)
- "24/7 SOC operations" (no SOC staff)
- "Round-the-clock monitoring" (without MSSP)

**Realistic:**
- "24/7 coverage via managed security services and automated alerting; [ORGANIZATION] provides escalation and coordination"
- "Automated 24/7 monitoring with [ORGANIZATION] escalation during business hours and on-call after hours"
- "24/7 threat detection via SIEM; [ORGANIZATION] response during business hours with escalation procedures for after-hours critical events"

---

## ROLE COMBINATION REALISM

### Appropriate Language for Small Organizations

**Include role combination clauses:**

"[ORGANIZATION] may combine roles (CISO, CIO/CTO, Privacy Officer, Compliance Officer, Records Officer) due to organizational size. Role conflicts are documented and mitigated. For larger or higher-risk contracts, [ORGANIZATION] may add personnel or subcontractors to separate these duties."

**Avoid:**
- Listing independent roles without acknowledging combination
- Implying full separation of duties when not achievable
- Creating unrealistic organizational charts

---

## STAFFING REALISM

### Appropriate Language for Small Organizations

**Include staffing augmentation clauses:**

"For contracts requiring 24/7 or multi-person coverage, [ORGANIZATION] may augment staffing with subcontractors or managed service providers, documented in the contract or task order."

**Placeholders:**
- "[To be designated]" is acceptable if immediately followed by augmentation clause
- Pre-identify partner organizations where possible
- Document in contract-specific plans

## APPLYING SCORING CRITERIA

### Consistency Guidelines

1. **Use the same criteria across all assessments** to enable comparison over time
2. **Document evidence** for each score assigned
3. **Be objective** - base scores on observable facts, not aspirations
4. **Consider context** - small organization maturity differs from enterprise
5. **Focus on gaps** - identify specific improvements needed, not just scores
6. **Be honest** - accurate assessment is more valuable than inflated scores
7. **Update regularly** - reassess as improvements are implemented

### Score Calibration

**Before finalizing scores:**
- Compare to scoring rubric descriptions
- Verify evidence supports the score
- Check for consistency across similar areas
- Validate against industry benchmarks for organization size
- Ensure language accuracy (compliant vs aligned)
- Confirm operational realism

## CONCLUSION

This scoring criteria provides a standardized, objective framework for assessing compliance documentation and implementation maturity. Use these criteria consistently to:

- Conduct package maturity assessments
- Perform comprehensive compliance reviews
- Evaluate cross-package dependencies
- Track improvement over time
- Prepare for external audits
- Make informed investment decisions

**Remember:** The goal is not to achieve a perfect score, but to maintain honest, accurate, and audit-ready documentation that reflects actual capabilities and supports business objectives.
