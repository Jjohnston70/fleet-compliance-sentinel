▶ What This Module Does

The Invoice Extraction Module takes fleet maintenance invoice PDFs and converts them into structured, normalized data. It automatically detects which vendor issued the invoice, applies the correct parser, and outputs clean records to Excel or JSON. Service types and maintenance categories are auto-classified from work descriptions so operations teams don't have to hand-label every invoice.

Problem

Manual PDF Data Entry

Staff spend hours copying invoice fields from PDFs into spreadsheets, introducing errors and inconsistency across vendors.

Solution

Automated Extraction

Drop PDFs in, get structured Excel or JSON out. Vendor-specific parsers handle each format automatically with validated field mapping.

Problem

No Audit Trail

No record of who processed which invoice, when, or what was extracted. Compliance gaps during audits.

Solution

SOC2-Compliant Logging

Every extraction writes a structured audit entry with operator, timestamp, org scoping, and PII masking. VINs and plate numbers never appear in logs.

☰ Excel Export Formats

Two export paths support different downstream workflows. Both produce TNDS-branded Excel files with frozen headers, money formatting, and auto-width columns.

Original Format

to_xlsx()
  • Invoices23 columns
  • Line Items11 columns

Matches maintenance_import.xlsx schema. Full vehicle detail with VIN, make, model, engine, mileage.

Fleet Compliance Format

to_fleet_xlsx()
  • Invoices15 columns
  • Maintenance Tracker13 columns
  • Line Items9 columns

Matches fleet-compliance-bulk-upload-template. Auto-derived category, service type, and maintenance IDs.

🔒 Compliance & Security

Audit Logging

Every extraction writes a structured JSON entry to audit.jsonl with operator, timestamp, org_id, vendor, and extraction status. No PII in logs.

PII Masking

VINs, plate numbers, customer names, and addresses are masked before any logging or external transmission. Masking is applied via compliance.mask_pii().

Input Validation

PDF files are validated for existence, magic bytes, size limits, and path traversal before processing. Rejects non-PDF files at the gate.

Org Scoping

Multi-tenant isolation via org_id tagging on all extracted records. scoped_extract() enforces org boundaries automatically.

Controls aligned with SOC2 Type II, HIPAA, CCPA/CPRA, and GLBA requirements. Data classification documented in DATA_CLASSIFICATION.md. All client data treated as regulated by default.
