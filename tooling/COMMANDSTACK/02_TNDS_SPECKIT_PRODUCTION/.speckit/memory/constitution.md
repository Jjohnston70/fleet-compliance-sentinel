## Governance

### Amendment Process
1. Propose change with rationale
2. Document migration plan for existing code
3. Require approval from [role/person]
4. Update version number per semantic versioning

### Version Policy
- **MAJOR**: Backward-incompatible principle changes
- **MINOR**: New principles or expanded guidance
- **PATCH**: Clarifications, typos, non-semantic refinements

### Compliance Review
- All PRs must verify constitution compliance
- Violations require explicit justification
- Unjustified violations block merge
