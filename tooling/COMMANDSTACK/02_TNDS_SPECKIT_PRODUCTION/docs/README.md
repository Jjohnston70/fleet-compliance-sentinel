# Documentation

This folder contains durable, shareable documentation.

## What Goes Here

- Deployment guides
- Architecture decisions
- Compliance checklists
- API documentation
- User guides
- Anything meant to be read later

## What Does NOT Go Here

- Work-in-progress drafts → `/notes`
- Scripts and tools → `/tools`
- Deprecated content → `/archive`
- Runtime config → root

## Naming Convention

Use descriptive names with UPPERCASE for major docs:
- `DEPLOYMENT_GUIDE.md`
- `ARCHITECTURE.md`
- `COMPLIANCE-CHECKLIST.md`
