# Archive

Frozen experiments and deprecated content. **Read-only by policy.**

## What Goes Here

- Completed experiments
- Deprecated approaches
- Old versions of major documents
- Abandoned features

## Rules

1. **Never edit files in archive** - they're historical record
2. **Add context** when archiving (why it was archived)
3. **Use folders** for related content: `archive/v1-auth-approach/`
4. **Date your archives** when relevant: `archive/2025-01-old-api/`

## What Does NOT Go Here

- Active documentation → `/docs`
- Work in progress → `/notes`
- Reusable scripts → `/tools`

## When to Archive

Move content here when:
- A decision is made (document the losing option)
- An experiment concludes (preserve learnings)
- A major refactor completes (keep the before state)
- A feature is abandoned (document why)

## Archive Entry Template

When adding content, include a brief `README.md`:

```markdown
# [Archive Name]

**Archived**: YYYY-MM-DD
**Reason**: [Why this was archived]
**Context**: [Brief description of what this was]
**Learnings**: [What we learned, if applicable]
```
