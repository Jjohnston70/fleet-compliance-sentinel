# ADVANCED COMPLIANCE USAGE GUIDE

## Purpose

This usage guide explains how, when, and why each document in the Advanced Compliance skill is used. This guide helps personnel understand the practical application of advanced compliance templates and specialized policies.

## Skill Overview

The Advanced Compliance skill contains 10 comprehensive documents that establish specialized compliance templates and advanced security policies. These documents are essential for:
- Client contracts and legal agreements
- Specialized compliance requirements (HIPAA, PCI DSS, etc.)
- Advanced security policies (API, remote work, mobile, supply chain)
- Audit readiness and compliance demonstrations
- Risk management and vendor assessments
- Situational compliance needs beyond core operations

This skill serves as the "advanced templates layer" that provides specialized compliance documentation for specific situations and requirements.

## Document Usage Guide

## Cross-Skill Integration

### Skill 2 (Security & Compliance Handbook):
- Skill 2 provides governance foundation; skill 7 provides specialized templates
- All skill 7 documents align with skill 2 requirements
- Skill 2 is comprehensive; skill 7 is situational

### Skill 1 (Internal Compliance):
- Skill 1 provides operational procedures; skill 7 provides specialized policies
- API Security (skill 7) references Logging Standard (skill 1)
- Remote Work and Mobile Device (skill 7) reference Access Control (skill 1)
- Supply Chain Security (skill 7) references Vendor Management (skill 1)

### Skill 3 (Data Handling & Privacy):
- Skill 3 provides data protection requirements; skill 7 templates implement them
- Data Processing Agreement (skill 7) references skill 3 data handling standards
- Business Associate Agreement (skill 7) references skill 3 HIPAA controls
- All skill 7 policies comply with no-PII-in-logs principle (skill 3)
- API Security, Remote Work, Mobile Device (skill 7) reference Anonymization Standard (skill 3)

### Skill 4 (Government Contracting):
- Skill 4 provides federal overlay; skill 7 provides general templates
- Audit Readiness Checklist (skill 7) includes federal compliance items (skill 4)
- Supply Chain Security (skill 7) supports federal procurement requirements (skill 4)

### Skill 5 (Cloud Platform Security):
- Skill 5 provides platform security; skill 7 provides specialized policies
- API Security (skill 7) applies to APIs on cloud (skill 5)
- Remote Work (skill 7) uses cloud workspace (skill 5)
- Mobile Device (skill 7) integrates with cloud endpoint management (skill 5)

### Skill 6 (Business Operations):
- Skill 6 provides business processes; skill 7 provides legal templates
- MSA and SLA templates (skill 7) govern client relationships (skill 6)
- Service offerings (skill 6) reference SLA commitments (skill 7)
- Security questionnaire (skill 7) references business capabilities (skill 6)

## Practical Usage Scenarios

### Scenario 1: New Client Contract Negotiation
**Client requires formal MSA and DPA for engagement**
- Response: Use MSA Template and Data Processing Agreement Template
- Integrate with: skill 3 privacy requirements, skill 2 security commitments
- Result: Comprehensive client contract with legal protections

### Scenario 2: Healthcare Client with PHI
**Healthcare client requires HIPAA BAA for PHI processing**
- Response: Use Business Associate Agreement Template
- Integrate with: skill 3 HIPAA controls, skill 2 security requirements
- Result: Compliant HIPAA BAA for healthcare engagement

### Scenario 3: Client Security Assessment
**Client sends comprehensive security questionnaire**
- Response: Use Security Questionnaire Response Template
- Reference: skill 2 Handbook, skill 5 cloud security, skill 4 federal readiness
- Result: Comprehensive security questionnaire response

### Scenario 4: SOC 2 Audit Preparation
**Preparing for SOC 2 Type II audit**
- Response: Use Audit Readiness Checklist
- Integrate with: skill 2 Handbook, skill 1 operational procedures, skill 3 privacy controls
- Result: Comprehensive audit preparation and successful audit

### Scenario 5: API Development for Client
**Developing API for client integration**
- Response: Use API Security Policy
- Integrate with: skill 1 SDLC, skill 3 no-PII-in-logs, skill 5 cloud security
- Result: Secure API development and deployment

## Maintenance and Updates

### Annual Review:
- All skill 7 documents reviewed annually
- Review includes legal changes, template improvements, lessons learned
- Review coordinated with skill 2 Handbook review

### Triggered Updates:
- Updated when regulations change (GDPR, HIPAA, etc.)
- Updated when legal counsel recommends changes
- Updated when client feedback requires improvements
- Updated when audit findings require updates
- Updated when new compliance requirements emerge

### Legal Review:
- All contract templates reviewed by legal counsel
- Templates updated based on legal advice
- Changes tracked and documented
- Personnel notified of template changes

## Conclusion

Skill 7 is the advanced compliance and specialized templates layer that provides situational documentation for specific compliance needs. These documents build on the foundation of skills 1, 2, and 3, providing specialized templates and policies for complex situations. Personnel should be familiar with these documents and use them when specialized compliance requirements arise.
