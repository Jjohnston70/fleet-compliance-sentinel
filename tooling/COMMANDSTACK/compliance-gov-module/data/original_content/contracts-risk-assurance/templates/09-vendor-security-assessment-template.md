VENDOR SECURITY ASSESSMENT TEMPLATE

Purpose

This Vendor Security Assessment Template provides [COMPANY_LEGAL_NAME] with a standardized approach for evaluating the security posture of vendors, suppliers, and third-party service providers. This template ensures vendors meet security requirements and do not introduce unacceptable risks.

Scope

This template applies to all [COMPANY_NAME] vendor security assessments. It applies to all personnel involved in vendor selection, management, and oversight.

Governance and Cross-Package References

This template integrates security and data protection requirements into vendor security assessments.

Universal [COMPANY_ABBREVIATION] Principles - Vendor Assessment Application:
Data-as-Regulated: Vendors handling client/customer data must treat all data as regulated by default
No Sensitive Data in Logs: Vendors must not store PII, PHI, PCI, CUI in logs, prompts, debugging output, screenshots, support tickets

Cross-Package Integration:
security-governance skill: Master governance document; vendors must meet [COMPANY_ABBREVIATION] security requirements
internal-compliance skill (Vendor Management Policy): Vendor management requirements and oversight
data-handling-privacy skill: Vendor data handling requirements
internal-compliance skill (Audit Logging and Monitoring Policy): Vendor logging requirements (no PII in logs)
data-handling-privacy skill (Anonymization Standard): Vendor anonymization requirements for logs and monitoring

For vendor management requirements, see internal-compliance skill (Vendor Management Policy).
For data handling requirements, see data-handling-privacy skill.
For logging requirements, see internal-compliance skill (Audit Logging and Monitoring Policy).
