# TNDS Prompt Engineering Skill

## What This Is
A guided system for creating, testing, and improving prompts using Multi-Armed Bandits methodology. No AI/ML expertise required.

## Who It's For
TNDS team members who need to build or improve prompts for Command modules, client work, or internal automation.

## Problems This Removes
- **Bad Prompt Tax:** Stop wasting time guessing which prompt will work
- **Emotional Tweaking:** No more "it broke so let's tweak it back"
- **Manual A/B Testing:** System learns which prompts work best automatically
- **Stale Prompts:** Continuously adapts as users and models change
- **Unclear Success:** Defines measurable metrics upfront

## What You Get

### Guided Workflows
Four complete pathways based on your need:
- **Create:** Build new prompts from scratch with 5-8 variations
- **Modify:** Fix or improve existing prompts systematically  
- **Evaluate:** Test and compare prompt performance scientifically
- **Retire:** Safely deprecate outdated prompts

### Non-Technical Language
Explains complex concepts (Thompson Sampling, reward functions, exploration/exploitation) in plain English with real examples.

### Production-Ready Templates
- Prompt arm documentation format
- Logging infrastructure setup
- Metrics tracking sheets
- Weight update schedules

### Decision Support
- When to use which prompt type
- How to define success metrics
- When to retire vs. improve
- How much testing is enough

## Scope

### In Scope

- Tracking performance metrics

## Quick Start

### If You Need to Create a New Prompt:
1. Start with Phase 1: Identify Intent
2. Answer 3-5 discovery questions
3. Create 5-8 prompt variations
4. Set up basic logging
5. Let the system learn which works best

### If You Need to Fix a Broken Prompt:
1. Start with Phase 2B: Modify Existing
2. Diagnose the specific issue
3. Create ONE variant addressing it
4. Add to testing pool
5. Monitor results for 1-2 weeks

### If You Need to Compare Options:
1. Start with Phase 2C: Evaluate Performance
2. Set up comparison test
3. Log results for 2-4 weeks
4. Review metrics and make decision
